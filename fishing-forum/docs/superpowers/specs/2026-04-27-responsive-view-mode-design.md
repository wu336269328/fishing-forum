# PC / 手机网页适配设计

## 目标

为钓友圈前端实现可在 PC 和手机之间切换的网页体验。默认情况下页面按设备宽度自动响应；用户也可以手动切换为 PC版或手机版，并在下次访问时保留选择。

## 视图模式

前端维护一个全局视图模式：

- `auto`：默认值。根据浏览器宽度自动决定布局。
- `desktop`：强制 PC版。页面保持桌面最大宽度、横向导航和桌面布局。
- `mobile`：强制手机版。页面使用窄屏容器、移动导航和底部快捷栏。

视图模式保存到 `localStorage`。页面启动时读取保存值，并通过根节点 class 暴露给全局 CSS 和组件使用。

## 布局与导航

顶部导航保留现有视觉风格，但增加视图切换控件：

- PC 宽屏：显示 logo、主导航、视图切换、登录/用户操作。
- 手机布局：显示 logo、菜单按钮、视图切换；主导航收进抽屉或折叠菜单。
- 手机版增加底部快捷栏：首页、论坛、发帖、消息/登录、我的。

切换控件使用“自动 / PC / 手机”三态，用户点击后立即生效。

## 页面适配范围

本次实现优先覆盖全站基础可用性：

- 全局容器：控制最大宽度、移动端边距、卡片间距，避免横向溢出。
- 首页：手机模式下单列展示帖子，隐藏或下沉侧边栏内容。
- 论坛页：筛选区改为可换行/纵向布局，搜索和排序在窄屏下占满宽度。
- 其他视图：通过全局移动端 CSS 适配表单、卡片、分页、对话框、图片和长文本。

不在本次范围内拆成两套页面，也不重写后端接口。

## 数据流

新增前端状态模块或组合式函数负责：

1. 读取 `localStorage` 中的视图模式。
2. 在用户切换时更新状态并写回 `localStorage`。
3. 根据当前模式给根节点设置 `data-view-mode` 或 class。
4. 在 `auto` 模式下仍由 CSS media query 决定最终布局。

## 错误处理

如果保存的模式值非法，回退到 `auto`。如果浏览器不支持 `localStorage`，页面仍使用 `auto` 模式，不阻塞渲染。

## 测试与验证

验证项：

- 前端构建通过。
- 手动检查 PC 宽屏、手机窄屏、桌面强制手机版、手机强制 PC版。
- 检查导航、底部快捷栏、首页、论坛页、登录态/未登录态入口。
- 确认 `.superpowers/` 视觉伴随临时文件不会进入提交。
